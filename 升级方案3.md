### 第一步：改造 App 端 `InputInjector.kt` (斩断 Shell 降级后路)

### 第二步：改造脱壳端 `Main.java` (移除 InputManager 与纯物理按键化)

在脱壳进程 `input-server.jar` 的源码中，必须彻底移除 `kernelFailCount` 降级机制和 `KeyEvent` 相关的类。

**1. 重写 `injectKey` 物理按键方法**

原先物理按键（如返回、Home）使用了 `new KeyEvent()`。现在需要直接向 `/dev/input/eventX` 写入 Linux 内核按键协议 (`EV_KEY`)：

Java

```
// 在你的 KernelTouchInjector (或者 Main.java) 中添加物理按键类型常量
private static final short EV_KEY = 0x01;

// 纯物理层的按键注入
public void injectKey(int linuxKeyCode) throws Exception {
    // 1. 发送按键按下
    sendEvent(EV_KEY, (short) linuxKeyCode, 1);
    sendEvent(EV_SYN, SYN_REPORT, 0);
    
    // 模拟人类按键按压停顿
    Thread.sleep(30 + (int)(Math.random() * 20));
    
    // 2. 发送按键抬起
    sendEvent(EV_KEY, (short) linuxKeyCode, 0);
    sendEvent(EV_SYN, SYN_REPORT, 0);
}
```

*(注意：此处传入的 keyCode 应该是 Linux 内核键值，例如 Linux 的 `KEY_BACK` 通常是 `158`，`KEY_HOME` 是 `102`，这与 Android 层面的 `KeyEvent.KEYCODE_BACK` 数值不同，需要在 Socket 接收端做一层简单的映射转换)*

**2. 彻底删除降级逻辑**

如果在写入 `/dev/input/eventX` 时发生 `IOException`，直接向上层抛出异常或记录日志，**绝对不要**去 catch 异常后调用反射的 `InputManager.getInstance().injectInputEvent()`。将相关的 `import android.hardware.input.InputManager;` 彻底从你的 Java 源码中删除，确保在编译层面就斩断对框架的依赖。

### 第三步：处理文本输入 (`inputText`) 的终极隐患

目前的 `InputInjector.kt` 中的 `inputText(text: String)` 使用的是 `Shell.cmd("input text ...")` 或者复制到剪贴板然后发送粘贴按键的 fallback。这两种方式在系统底层都会被识别为框架层注入。

要实现 100% 纯正的内核级操控，你有两个选择：

- **方案 A（最安全，建议采用）：** 废弃 `inputText` 方法。强制主控端用户只能通过模拟真实点击手机屏幕上的“软键盘（虚拟键盘）”来输入文字。这等同于物理手指在敲击屏幕，防检测级别最高。
- **方案 B（极难）：** 将传入的字符串（如果是英文字符），逐个翻译成对应的 Linux 物理键盘键值 (`KEY_A`, `KEY_B` 等)，然后通过上面重写的 `injectKey` 方法发送。这需要设备节点挂载了物理键盘属性，且不支持直接输入中文。

建议将 `inputText` 方法清空或加上强制警告：

Kotlin

```
    fun inputText(text: String) {
        Log.e(TAG, "纯内核级注入模式下，禁止使用系统框架级的 input text 命令。请通过模拟点击屏幕软键盘输入文本。")
    }
```

### 🚨 致命 Bug 1：时间戳“时空错乱” (The Timeval Trap)

**问题代码：**

Java

```
long time = System.currentTimeMillis(); // 这是现实世界的绝对时间 (墙上时钟)
long sec = time / 1000;
long usec = (time % 1000) * 1000;
buffer.putLong(sec); buffer.putLong(usec);
```

**原理解析：**

Android 框架层的 `InputDispatcher` 校验输入事件时，使用的是 **`CLOCK_MONOTONIC`（系统开机运行时间，即 `SystemClock.uptimeMillis()`）**。而你写入内核的是 **`CLOCK_REALTIME`（现实绝对时间）**。

两者的时间差可能高达几十年！当这串数据被 Android 底层读取时，系统会认为这是一个“来自未来的错误事件”或者计算滑动速度时出现负数/无限大，导致事件被直接丢弃或滑动异常。

**完美解法：**

Linux 内核非常聪明，当你向 `/dev/input/eventX` 写入时间戳为 `0` 的事件时，内核的 `input core` 会自动帮你打上最精确的当前底层 Monotonic 时间戳。

修复：直接填 `0` 即可。

Java

```
if (is64Bit) {
    buffer.putLong(0); // 直接写 0，内核会自动填充正确的时间！
    buffer.putLong(0);
} else {
    buffer.putInt(0);
    buffer.putInt(0);
}
```

### 🚨 致命 Bug 2：抬手时的“(0,0) 瞬间移动” (Teleportation Bug)

**问题代码：** 在 `injectTap`、`injectSwipe`、`injectLongPress` 的抬起阶段（UP）：

Java

```
sendEvent(EV_ABS, ABS_X, 0); // 致命错误！
sendEvent(EV_ABS, ABS_Y, 0); // 致命错误！
```

**原理解析：**

当你手指离开屏幕（ACTION_UP）时，你向内核发送了坐标 `(0,0)`。这会告诉 Android 系统：“我的手指在离开屏幕前的一瞬间，以光速瞬移到了屏幕左上角(0,0)”。

这会导致极其严重的后果：

1. **幽灵误触**：原本点击屏幕中央，松手时却触发了屏幕左上角返回按钮的点击。
2. **滑动速度爆炸**：向右滑动时，最后瞬间坐标归零，底层计算出的滑动加速度无穷大，导致列表直接飞到顶部或底部。

**完美解法：**

在模拟抬手（UP）时，**绝不允许发送任何坐标变化**。只需告诉内核追踪 ID 失效 (`-1`)，压力归零，按键抬起即可。

**修复：** 删除 UP 阶段的 `ABS_X` 和 `ABS_Y` 发送。

### 🚨 致命 Bug 3：物理按键写入了错误的硬件节点

**问题代码：**

Java

```
public void injectKey(int keyCode) {
    sendEvent(EV_KEY, (short) keyCode, 1); // 发送物理按键
}
```

**原理解析：**

你正在向 `touchNode`（触摸屏节点，通常是 `event2`）写入 `KEY_BACK` 或 `KEY_HOME` 等物理按键事件。

在 Android 底层的 `EventHub.cpp` 中，系统在开机时会读取每个节点的“能力位图 (Capability Bitmask)”。触摸屏节点的位图里只声明了它支持 `BTN_TOUCH`。当你把 `KEY_BACK` 塞进触摸屏节点时，`EventHub` 发现该设备不支持这个键，会**直接丢弃**，导致你的按键操作完全无效。

**完美解法：**

做纯内核级控制，**不要尝试用硬件节点注入物理按键**（除非你去寻找 `gpio-keys` 节点）。

现代 Android 手机全都支持“全面屏手势”。如果你想实现 BACK（返回）和 HOME（回到桌面），应该**改用 Swipe（滑动）来代替按键**！

- **BACK 返回**：从屏幕左边缘 (0, y) 向右滑动到 (200, y)。
- **HOME 桌面**：从屏幕最底端 (x, screenHeight) 向上滑动到 (x, screenHeight - 400)。

### ⚠️ 隐患 1：`ABS_MT_TRACKING_ID` 随机碰撞

**问题代码：**

Java

```
sendEvent(EV_ABS, ABS_MT_TRACKING_ID, (int)(Math.random() * 10000) + 1);
```

**原理解析：**

Linux `Input Multi-Touch` 协议要求 `Tracking ID` 最好是一个单调递增的唯一标识符。虽然随机数有概率能用，但如果连续两次生成了相同的数字，或者生成的数字碰巧和用户正在真实触摸屏幕的手指 ID 撞车，系统会导致触摸流错乱。

**完美解法：**

使用一个静态递增计数器。

Java

```
// 在类中声明静态变量
private static int trackingIdCounter = 1;

// 使用时递增并取模防止溢出
int trackingId = (trackingIdCounter++) % 10000 + 1;
sendEvent(EV_ABS, ABS_MT_TRACKING_ID, trackingId);
```

```
    static class KernelTouchInjector {
        private final FileOutputStream out;
        private final boolean is64Bit;
        private final int structSize;
        private final byte[] bufferArray;
        private final ByteBuffer buffer;
        private static int trackingIdCounter = 1;

        private static final short EV_SYN = 0x00;
        private static final short EV_KEY = 0x01;
        private static final short EV_ABS = 0x03;

        private static final short SYN_REPORT = 0x00;
        private static final short BTN_TOUCH = 0x14a;
        private static final short ABS_X = 0x00;
        private static final short ABS_Y = 0x01;
        private static final short ABS_MT_SLOT = 0x2f;
        private static final short ABS_MT_TRACKING_ID = 0x39;
        private static final short ABS_MT_POSITION_X = 0x35;
        private static final short ABS_MT_POSITION_Y = 0x36;
        private static final short ABS_MT_PRESSURE = 0x3a;
        private static final short ABS_MT_TOUCH_MAJOR = 0x30;

        public KernelTouchInjector(String deviceNode) throws Exception {
            this.is64Bit = android.os.Process.is64Bit();
            this.structSize = is64Bit ? 24 : 16;
            this.bufferArray = new byte[structSize];
            this.buffer = ByteBuffer.wrap(bufferArray).order(ByteOrder.LITTLE_ENDIAN);
            this.out = new FileOutputStream(deviceNode);
        }

        private synchronized void sendEvent(short type, short code, int value) throws Exception {
            buffer.clear();
            // 修复1: 时间戳全部填0，让 Linux 内核 input core 自动填入最精确的 Monotonic 时间
            if (is64Bit) {
                buffer.putLong(0L); 
                buffer.putLong(0L);
            } else {
                buffer.putInt(0);
                buffer.putInt(0);
            }
            buffer.putShort(type);
            buffer.putShort(code);
            buffer.putInt(value);

            out.write(bufferArray);
        }

        private int getNextTrackingId() {
            return (trackingIdCounter++) % 10000 + 1;
        }

        public void injectTap(int x, int y) throws Exception {
            int jitterX = x + (int)(Math.random() * 5 - 2);
            int jitterY = y + (int)(Math.random() * 5 - 2);
            int trackingId = getNextTrackingId();

            // Down
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, trackingId);
            sendEvent(EV_ABS, ABS_MT_POSITION_X, jitterX);
            sendEvent(EV_ABS, ABS_MT_POSITION_Y, jitterY);
            sendEvent(EV_ABS, ABS_X, jitterX);
            sendEvent(EV_ABS, ABS_Y, jitterY);
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 50);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 10);
            sendEvent(EV_KEY, BTN_TOUCH, 1);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();

            Thread.sleep(45 + (int)(Math.random() * 20));

            // Up
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, -1);
            // 修复2: 绝对不能在这里发 ABS_X=0 和 ABS_Y=0，直接清空压力和触控状态即可
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 0);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 0);
            sendEvent(EV_KEY, BTN_TOUCH, 0);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();
        }

        public void injectSwipe(int x1, int y1, int x2, int y2, int durationMs) throws Exception {
            int steps = Math.max(durationMs / 16, 1);
            int stepTime = durationMs / steps;
            int trackingId = getNextTrackingId();

            // Down
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, trackingId);
            sendEvent(EV_ABS, ABS_MT_POSITION_X, x1);
            sendEvent(EV_ABS, ABS_MT_POSITION_Y, y1);
            sendEvent(EV_ABS, ABS_X, x1);
            sendEvent(EV_ABS, ABS_Y, y1);
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 50);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 10);
            sendEvent(EV_KEY, BTN_TOUCH, 1);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();

            // Move
            for (int i = 1; i <= steps; i++) {
                Thread.sleep(stepTime);
                int cx = x1 + (x2 - x1) * i / steps;
                int cy = y1 + (y2 - y1) * i / steps;

                sendEvent(EV_ABS, ABS_MT_POSITION_X, cx);
                sendEvent(EV_ABS, ABS_MT_POSITION_Y, cy);
                sendEvent(EV_ABS, ABS_X, cx);
                sendEvent(EV_ABS, ABS_Y, cy);
                sendEvent(EV_SYN, SYN_REPORT, 0);
                out.flush();
            }

            Thread.sleep(stepTime);
            
            // Up
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, -1);
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 0);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 0);
            sendEvent(EV_KEY, BTN_TOUCH, 0);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();
        }

        public void injectLongPress(int x, int y) throws Exception {
            int trackingId = getNextTrackingId();

            // Down
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, trackingId);
            sendEvent(EV_ABS, ABS_MT_POSITION_X, x);
            sendEvent(EV_ABS, ABS_MT_POSITION_Y, y);
            sendEvent(EV_ABS, ABS_X, x);
            sendEvent(EV_ABS, ABS_Y, y);
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 50);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 10);
            sendEvent(EV_KEY, BTN_TOUCH, 1);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();

            Thread.sleep(600);

            // Up
            sendEvent(EV_ABS, ABS_MT_SLOT, 0);
            sendEvent(EV_ABS, ABS_MT_TRACKING_ID, -1);
            sendEvent(EV_ABS, ABS_MT_PRESSURE, 0);
            sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 0);
            sendEvent(EV_KEY, BTN_TOUCH, 0);
            sendEvent(EV_SYN, SYN_REPORT, 0);
            out.flush();
        }

        // 废弃原有的硬件物理按键注入，强制主控端改用手势滑动！

        }
```
