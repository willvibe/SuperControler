我们要实现 **“内核驱动层注入 (Kernel/Driver Level Injection)”**。现在要利用root权限**下探到 Linux 内核驱动层**。

基于你目前极为优秀的 `主应用 <-> LocalSocket <-> 幽灵进程 (Root)` 架构，我们完全不需要动外层的网络和 UI。**只需要修改运行在 Root 权限下的 `core/input-server` 模块即可。**

下面是为你量身定制的**直接向 `/dev/input/eventX` 写入二进制数据**的详细改造方案，纯 Java 实现，无需配置 NDK/C++。

### 第一步：动态寻找物理触屏的设备节点

Android 手机内部有多个输入设备（按键、陀螺仪、距离传感器等），触屏通常是 `/dev/input/event2` 或 `event3`。我们需要在 `input-server` 启动时自动找到它。

在你的 `Main.java` (幽灵进程入口) 中，加入以下方法来寻找触屏节点：

Java

```
import java.io.BufferedReader;
import java.io.InputStreamReader;

public static String findTouchScreenNode() {
    try {
        // 执行 getevent -pl 寻找包含 ABS_MT_POSITION_X 的硬件节点
        Process process = Runtime.getRuntime().exec("getevent -pl");
        BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()));
        String line;
        String currentNode = null;
        
        while ((line = reader.readLine()) != null) {
            if (line.startsWith("add device")) {
                // 解析节点路径，例如: add device 1: /dev/input/event2
                currentNode = line.substring(line.indexOf("/dev/input/event"));
            } else if (line.contains("ABS_MT_POSITION_X")) {
                // 只要这个设备支持多点触控的 X 坐标，它就是我们要的物理触屏
                return currentNode;
            }
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
    return "/dev/input/event2"; // 降级默认值
}
```

### 第二步：编写 Linux `input_event` 的二进制写入器

Linux 内核接收的输入指令是一个个 C 语言的 `struct input_event`。现在的 Android 手机基本都是 64 位，结构体大小为 **24 字节**。我们需要用 Java 的 `ByteBuffer` 将坐标和状态打包成二进制字节流并直接写入。

新建或在 `Main.java` 内部创建一个类：

Java

```
import java.io.FileOutputStream;
import java.nio.ByteBuffer;
import java.nio.ByteOrder;

public class KernelTouchInjector {
    private FileOutputStream out;
    
    // Linux Event 常量
    private static final short EV_SYN = 0x00;
    private static final short EV_KEY = 0x01;
    private static final short EV_ABS = 0x03;
    
    private static final short SYN_REPORT = 0x00;
    private static final short BTN_TOUCH = 0x14a; // 330
    private static final short ABS_MT_SLOT = 0x2f; // 47
    private static final short ABS_MT_TRACKING_ID = 0x39; // 57
    private static final short ABS_MT_POSITION_X = 0x35; // 53
    private static final short ABS_MT_POSITION_Y = 0x36; // 54

    public KernelTouchInjector(String deviceNode) throws Exception {
        // 因为这是 Root 幽灵进程，可以直接打开硬件节点
        this.out = new FileOutputStream(deviceNode);
    }

    // 将数据打包为 64 位 Linux 内核规范的 input_event 结构
    private void sendEvent(short type, short code, int value) throws Exception {
        // 64 位系统下 struct input_event 为 24 字节
        ByteBuffer buffer = ByteBuffer.allocate(24);
        buffer.order(ByteOrder.LITTLE_ENDIAN); // 底层必须是小端序
        
        long time = System.currentTimeMillis();
        long sec = time / 1000;
        long usec = (time % 1000) * 1000;
        
        // struct timeval (16 bytes on 64-bit)
        buffer.putLong(sec);
        buffer.putLong(usec);
        
        // type, code, value (8 bytes)
        buffer.putShort(type);
        buffer.putShort(code);
        buffer.putInt(value);
        
        out.write(buffer.array());
    }

    public void injectTap(int x, int y) throws Exception {
        // 【仿生学防封策略】坐标微小随机偏移 (Jitter)
        // 避开每次都点在绝对完美的中心像素上
        int jitterX = x + (int)(Math.random() * 5 - 2); 
        int jitterY = y + (int)(Math.random() * 5 - 2);

        // 1. 手指按下 (ACTION_DOWN)
        sendEvent(EV_ABS, ABS_MT_SLOT, 0); // 使用 Slot 0
        sendEvent(EV_ABS, ABS_MT_TRACKING_ID, (int)(Math.random() * 10000) + 1); // 分配随机触摸 ID
        sendEvent(EV_ABS, ABS_MT_POSITION_X, jitterX);
        sendEvent(EV_ABS, ABS_MT_POSITION_Y, jitterY);
        sendEvent(EV_KEY, BTN_TOUCH, 1);   // 屏幕感应到触摸
        sendEvent(EV_SYN, SYN_REPORT, 0);  // 告诉内核：这帧数据打包完毕，触发中断！

        // 2. 仿生学延迟 (模拟真实人类手指停留在屏幕上的时间)
        // 人类最快点击也需要 30ms-60ms，设为 0 会被秒封！
        Thread.sleep(45 + (int)(Math.random() * 20));

        // 3. 手指抬起 (ACTION_UP)
        sendEvent(EV_ABS, ABS_MT_SLOT, 0);
        sendEvent(EV_ABS, ABS_MT_TRACKING_ID, -1); // 释放触摸 ID
        sendEvent(EV_KEY, BTN_TOUCH, 0);           // 屏幕触摸消失
        sendEvent(EV_SYN, SYN_REPORT, 0);          // 告诉内核打包完毕

        out.flush();
    }
}
```

### 第三步：集成到你的 `LocalSocket` 通信框架中

在 `core/input-server` 模块的 `Main.java` 中（也就是接收 Socket 指令的循环处理逻辑里），将旧的注入方法替换为我们的内核级方法：

Java

```
public class Main {
    public static void main(String[] args) {
        // ... Socket 初始化与身份校验逻辑 ...

        try {
            // 1. 初始化内核注入器
            String touchNode = findTouchScreenNode();
            KernelTouchInjector injector = new KernelTouchInjector(touchNode);
            System.out.println("Kernel Touch Injector initialized on: " + touchNode);

            // 2. 循环读取主应用通过 Socket 发来的短指令
            BufferedReader reader = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));
            String line;
            while ((line = reader.readLine()) != null) {
                String[] parts = line.split(",");
                String cmd = parts[0];

                try {
                    if (cmd.equals("T")) { // 处理 Tap
                        int x = Integer.parseInt(parts[1]);
                        int y = Integer.parseInt(parts[2]);
                        
                        // 摒弃掉以前的 InputManager / Shell.cmd
                        // 换成内核驱动写入
                        injector.injectTap(x, y);
                    }
                    // TODO: 可以用同样的原理扩展 "S" (Swipe) 滑动操作的内核写入
                } catch (Exception e) {
                    System.err.println("Inject error: " + e.getMessage());
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
