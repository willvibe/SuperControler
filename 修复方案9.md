1. 经过对主控端（Controller）相关代码的深度排查，主控端在点击“连接”后立即发生闪退，**根本原因在于 WebRTC Native 层（C++ JNI）初始化顺序倒置，以及 EGL 上下文被异常销毁导致的底层严重内存段错误（SIGSEGV / SIGABRT）。**

   以下是导致闪退的三个致命 Bug 及详细修复方案：

   ### 🚨 致命 Bug 1：WebRTC Native 库未加载即被调用 (必现闪退)

   **所在文件:** `WebRtcClient.kt`

   **分析:** 在代码中，`EglBase` 是作为类属性被**立即初始化**的：

   Kotlin

   ```
   private val eglBase: EglBase = EglBase.create() // 👈 实例化时立即执行
   ```

   当你在 `ControllerService` 中点击连接并触发 `initWebRtcAsController` 时，会执行 `val client = WebRtcClient(...)`。此时类被实例化，立刻执行 `EglBase.create()` 尝试创建 EGL 上下文。

   **致命点在于**：此时 WebRTC 的底层 C++ 库（`libjingle_peerconnection_so.so`）根本还没有被初始化！WebRTC 规定必须先执行 `PeerConnectionFactory.initialize()` 才能调用任何 EglBase 相关逻辑，顺序颠倒会直接抛出 `UnsatisfiedLinkError` 或底层 Native 闪退。

   ### 🚨 致命 Bug 2：重连/断开时 EGL 僵尸上下文崩溃 (SIGSEGV)

   **所在文件:** `WebRtcClient.kt` & `ControllerActivity.kt`

   **分析:** 在 `WebRtcClient.dispose()` 方法中，你调用了 `eglBase.release()` 销毁了环境。

   然而，主界面的 `SurfaceViewRenderer` 仍然**死死咬住**这个旧的 EglBase 上下文。如果用户断线重连，服务器下发新的流，当新的 `VideoTrack` 试图将视频帧渲染到这个**已被销毁**的旧 EGL 上下文时，会导致 C++ 渲染线程出现非法内存访问，应用瞬间强制闪退，没有任何 Java 层报错。

   ### 🚨 致命 Bug 3：渲染器重复初始化异常

   **所在文件:** `ControllerActivity.kt`

   **分析:**

   在 `connectToDevice()` 中，重置了 `surfaceInitialized = false`，但**没有**调用 `surfaceViewRenderer.release()`。当尝试重新拉流时，它会再次调用 `surfaceViewRenderer.init(...)`。对一个未释放的渲染器重复调用 init，会直接抛出 `IllegalStateException("Already initialized")`。

   ------

   ### 🛠️ 详细修复方案

   #### 第一步：改造 `WebRtcClient.kt`（实现安全的全局单例 EGL）

   我们需要将 `PeerConnectionFactory` 的初始化和 `EglBase` 改造成全局安全的单例，并严格控制初始化顺序。

   1. **替换全局静态区与成员变量**

      将伴生对象（Companion object）和顶部的 EglBase 变量修改为如下：

   Kotlin

   ```
       companion object {
           private const val TAG = "WebRtcClient"
           private const val VIDEO_TRACK_ID = "ScreenCapture"
           private const val DATA_CHANNEL_LABEL = "ControlChannel"
   
           // 【修复1】确保 Native 库只初始化一次
           @Volatile
           private var isFactoryInitialized = false
   
           // 【修复2】全局共享 EGL 上下文，防止 SurfaceViewRenderer 拿到死锁的上下文
           @Volatile
           private var sharedEglBase: EglBase? = null
   
           fun getSharedEglBase(context: Context): EglBase {
               if (!isFactoryInitialized) {
                   val initOptions = PeerConnectionFactory.InitializationOptions
                       .builder(context.applicationContext)
                       .setEnableInternalTracer(false)
                       .createInitializationOptions()
                   PeerConnectionFactory.initialize(initOptions)
                   isFactoryInitialized = true
               }
               if (sharedEglBase == null) {
                   sharedEglBase = EglBase.create()
               }
               return sharedEglBase!!
           }
       }
   
       // 【修复3】移除立即初始化的 val，改为可空的安全类型
       private var eglBase: EglBase? = null
       val eglBaseContext: EglBase.Context? get() = eglBase?.eglBaseContext
   ```

   1. **修改 `initialize()` 方法**

      在初始化工厂时，安全地获取 EglBase：

   Kotlin

   ```
       fun initialize() {
           if (isDisposed) return
           Log.i(TAG, "initialize()")
   
           // 【修复4】严格保证顺序：先 init Native 库，再获取 EglBase
           eglBase = getSharedEglBase(context)
   
           val encoderFactory = DefaultVideoEncoderFactory(eglBase!!.eglBaseContext, true, true)
           val decoderFactory = DefaultVideoDecoderFactory(eglBase!!.eglBaseContext)
   
           peerConnectionFactory = PeerConnectionFactory.builder()
               .setVideoEncoderFactory(encoderFactory)
               .setVideoDecoderFactory(decoderFactory)
               .createPeerConnectionFactory()
   
           Log.i(TAG, "PeerConnectionFactory created")
       }
   ```

   1. **修改 `dispose()` 方法**

      在 `dispose()` 方法的最底部，**删除对 `eglBase.release()` 的调用**：

   Kotlin

   ```
           queuedRemoteIceCandidates.clear()
           queuedLocalIceCandidates.clear()
   
           // ❌ 必须删除这段代码！绝对不能释放全局共享的 EGL Context，否则 UI 渲染器必崩
           // try {
           //     eglBase.release()
           // } catch (_: Exception) {}
   
           isScreenCaptureReady = false
   ```

   #### 第二步：修复 `ControllerActivity.kt`（解决渲染器重连崩溃）

   1. **修改 `connectToDevice()` 方法**

      在断开旧连接或开启新连接前，必须彻底清理并释放旧的渲染器状态。

   Kotlin

   ```
       private fun connectToDevice() {
           Log.i(TAG, "connectToDevice() targetId=$targetId")
           updateStatus("正在连接服务器...")
   
           // 【修复5】重新连接前，必须彻底释放旧的渲染器
           if (surfaceInitialized) {
               try {
                   surfaceViewRenderer.release()
               } catch (e: Exception) {
                   Log.w(TAG, "Surface release error: ${e.message}")
               }
           }
   
           isConnected = false
           surfaceInitialized = false
           isSurfaceReady = false
           surfaceInitAttempts = 0
           currentVideoTrack?.removeSink(surfaceViewRenderer)
           currentVideoTrack = null
           pendingVideoTrack?.removeSink(surfaceViewRenderer)
           pendingVideoTrack = null
           
           // ... 原有的 ControllerService 相关代码保持不变 ...
   ```

   1. **修改 `tryInitSurface()` 方法**

      由于上一步我们将 `eglBaseContext` 改为了可空类型，需要处理判空。

   Kotlin

   ```
       private fun tryInitSurface() {
           if (surfaceInitialized) return
           if (surfaceInitAttempts > 50) {
               Log.e(TAG, "Surface init failed after 50 attempts, giving up")
               return
           }
           surfaceInitAttempts++
           
           // 【修复6】处理安全的可空 EglBaseContext
           val eglCtx = webRtcClient?.eglBaseContext
           if (eglCtx != null) {
               doInitSurface(eglCtx)
               return
           }
           surfaceViewRenderer.postDelayed({ tryInitSurface() }, 300)
       }
   ```
