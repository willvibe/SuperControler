1. 发现一个非常经典的**资源泄漏（Resource Leak）\**与\**UI渲染过载**导致的问题。

   你遇到的情况可以归结为以下两个核心原因：

   1. **为什么一小会儿就断，且一直显示网络中断（重启却能秒连）？**
      - **OkHttpClient 线程与连接池泄漏**：在之前的代码中，每次断线超过一定次数（看门狗触发）就会 `new` 一个新的 `SignalingClient`。而在你的 `SignalingClient` 中，`okHttpClient` 是一个**实例变量**。这意味着每次重连都会创建一个全新的 OkHttp 引擎！几分钟内就会堆积几十个连接池和几百个后台线程，瞬间耗尽手机的底层 Socket 资源，导致系统强制拒绝后续网络请求（抛出 `Connection reset` 或网络中断）。重启 App 释放了内存，所以马上又能连上。
      - **心跳“自杀”漏洞**：你的 `startAppHeartbeat` 协程中，如果遇到一次底层的发送缓冲满（`ws.send()` 返回 `false`），代码直接 `break` 退出了心跳循环。导致后续再也没有心跳包发给服务端，45秒后触发强制断网。
   2. **首页越来越卡？**
      - **日志无限叠加**：`TextView` 中的文本如果无限叠加（几千行），每次 `append` 都会触发系统重新测量和绘制整个庞大的文本块，极大地拖慢主线程（UI卡顿），甚至导致 OOM（内存溢出）。

   下面是针对这两个文件的终极完美修复方案：

   ### 🛠️ 第一步：修复网络泄漏与心跳断连漏洞

   打开 `core/network/src/main/java/com/yourapp/remotectrl/network/SignalingClient.kt`。

   **1. 将 OkHttpClient 移为全局单例（彻底解决泄漏）**

   找到类最顶部的 `companion object` 和原本的 `createOkHttpClient` 方法。修改如下：

   Kotlin

   ```
       companion object {
           private const val FALLBACK_IP = "74.48.6.172"
           private const val FALLBACK_PORT = "8765"
           private const val TAG = "Signaling"
   
           // 【核心修复1】：将 OkHttpClient 改为伴生对象全局单例，所有 SignalingClient 共用一个引擎
           // 彻底杜绝看门狗重启服务时造成的线程池和 Socket 句柄泄漏！
           private val sharedOkHttpClient: OkHttpClient by lazy {
               val builder = OkHttpClient.Builder()
                   .pingInterval(45, TimeUnit.SECONDS)
                   .readTimeout(0, TimeUnit.MILLISECONDS)
                   .writeTimeout(15, TimeUnit.SECONDS)
                   .connectTimeout(20, TimeUnit.SECONDS)
                   .retryOnConnectionFailure(true)
                   .dns(object : Dns {
                       override fun lookup(hostname: String): List<InetAddress> {
                           try {
                               val addresses = Dns.SYSTEM.lookup(hostname)
                               val ipv4Only = addresses.filter { it is Inet4Address }
                               return if (ipv4Only.isNotEmpty()) ipv4Only else addresses
                           } catch (e: Exception) {
                               Log.e(TAG, "DNS lookup failed for $hostname: ${e.message}")
                               throw e
                           }
                       }
                   })
   
               try {
                   val trustAllCerts = arrayOf<javax.net.ssl.TrustManager>(
                       object : javax.net.ssl.X509TrustManager {
                           override fun checkClientTrusted(chain: Array<java.security.cert.X509Certificate>, authType: String) {}
                           override fun checkServerTrusted(chain: Array<java.security.cert.X509Certificate>, authType: String) {}
                           override fun getAcceptedIssuers(): Array<java.security.cert.X509Certificate> = arrayOf()
                       }
                   )
                   val sslContext = javax.net.ssl.SSLContext.getInstance("SSL")
                   sslContext.init(null, trustAllCerts, java.security.SecureRandom())
                   builder.sslSocketFactory(sslContext.socketFactory, trustAllCerts[0] as javax.net.ssl.X509TrustManager)
                   builder.hostnameVerifier { _, _ -> true }
               } catch (e: Exception) {
                   Log.e(TAG, "Failed to configure TLS: ${e.message}")
               }
               builder.build()
           }
       }
   
       // 删除原先的 private val okHttpClient = createOkHttpClient() 和 createOkHttpClient() 方法
   ```

   **3. 移除心跳自杀逻辑**

   找到 `startAppHeartbeat()` 方法，**删除两个 `break`**，即使当前帧发送失败也应该坚挺地等待下一次！  

   

   Kotlin

   ```
       private fun startAppHeartbeat(ws: WebSocket) {
           heartbeatJob?.cancel()
           heartbeatJob = scope.launch {
               while (isActive && !isDestroyed) {
                   delay(20_000)
                   try {
                       val ping = JSONObject().apply { put("type", "ping") }
                       val sent = ws.send(ping.toString())
                       if (!sent) {
                           Log.w(TAG, "Heartbeat send failed (buffer full or closing), keeping loop alive")
                           // 【核心修复3】：绝对不能 break！如果这里断了，健康检查一定会在45秒后因为没有心跳被踢下线！
                       }
                   } catch (e: Exception) {
                       Log.w(TAG, "Heartbeat error: ${e.message}")
                       // 【核心修复3】：同样不能 break！
                   }
               }
           }
       }
   ```

   **2. 彻底改造 `appendLog()` 方法**

   Kotlin

   ```
       private fun appendLog(message: String) {
           val timestamp = java.text.SimpleDateFormat("HH:mm:ss", java.util.Locale.getDefault())
               .format(java.util.Date())
           
           runOnUiThread {
               // 【核心修复】：使用双向链表管理，超限自动剔除最老的日志，然后全量渲染
               logList.add("[$timestamp] $message")
               if (logList.size > MAX_LOG_LINES) {
                   logList.removeFirst()
               }
               // 使用 joinToString 组合文本，极大降低内存占用和 TextView 渲染压力
               logText.text = logList.joinToString("\n") + "\n"
               
               // 滚动到底部
               logScrollView.post { logScrollView.fullScroll(View.FOCUS_DOWN) }
           }
       }
   ```
