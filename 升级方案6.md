针对将现有的“直接写 `/dev/input/eventX`”升级为“使用 Linux 标准虚拟输入子系统 (`uinput`)”的方案，由于 Java 原生不支持直接调用 Linux 的 `ioctl`（输入输出控制宏），我们需要引入 **JNI (Java Native Interface) 和 C++** 来完成底层改造。

这种升级将使你的 `SuperControler` 在内核眼中成为一个“合法的硬件设备”，从而解决时间戳撕裂和上下文溯源问题。

以下是基于你的 `core/input-server` 模块的具体代码重构升级方案：

### 升级架构概览

1. **新增 Native 层**：使用 C/C++ 编写一个 `.so` 动态链接库，负责打开 `/dev/uinput` 并向内核注册一个虚拟的触摸屏设备。
2. **重构 Java 负载**：将 `Main.java` 中的 `KernelTouchInjector` 替换为调用 JNI 方法的 `UinputInjector`。
3. **构建系统调整**：在 `input-server` 模块中配置 CMake 编译 C++ 代码，并将 `.so` 库与 `input-server.jar` 一并推送到受控端。

### 第一步：编写核心 C++ 驱动桥接 (`uinput_bridge.cpp`)

在 `core/input-server/src/main/cpp/` 目录下（你需要新建这个目录），创建 `uinput_bridge.cpp`。这段代码利用内核 API 1:1 伪造一个真实的触控板：

C++

```
#include <jni.h>
#include <string.h>
#include <fcntl.h>
#include <unistd.h>
#include <linux/input.h>
#include <linux/uinput.h>
#include <android/log.h>

#define LOG_TAG "UinputBridge"
#define LOGI(...) __android_log_print(ANDROID_LOG_INFO, LOG_TAG, __VA_ARGS__)
#define LOGE(...) __android_log_print(ANDROID_LOG_ERROR, LOG_TAG, __VA_ARGS__)

int uinput_fd = -1;

// 注册虚拟设备
extern "C" JNIEXPORT jboolean JNICALL
Java_com_yourapp_remotectrl_input_UinputInjector_nativeInit(JNIEnv *env, jobject thiz, jint width, jint height) {
    uinput_fd = open("/dev/uinput", O_WRONLY | O_NONBLOCK);
    if (uinput_fd < 0) {
        LOGE("Failed to open /dev/uinput");
        return JNI_FALSE;
    }

    // 1. 设置设备支持的事件类型 (EV_KEY, EV_ABS, EV_SYN)
    ioctl(uinput_fd, UI_SET_EVBIT, EV_KEY);
    ioctl(uinput_fd, UI_SET_EVBIT, EV_ABS);
    ioctl(uinput_fd, UI_SET_EVBIT, EV_SYN);

    // 2. 设置按键属性 (支持手指触摸标识)
    ioctl(uinput_fd, UI_SET_KEYBIT, BTN_TOUCH);
    ioctl(uinput_fd, UI_SET_KEYBIT, BTN_TOOL_FINGER);

    // 3. 设置绝对坐标轴属性 (支持多点触控协议 B)
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_X);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_Y);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_SLOT);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_TRACKING_ID);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_POSITION_X);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_POSITION_Y);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_PRESSURE);
    ioctl(uinput_fd, UI_SET_ABSBIT, ABS_MT_TOUCH_MAJOR);

    // 4. 配置虚拟设备的硬件信息和坐标范围
    struct uinput_user_dev uidev;
    memset(&uidev, 0, sizeof(uidev));
    snprintf(uidev.name, UINPUT_MAX_NAME_SIZE, "SuperControler_Virtual_Touch");
    uidev.id.bustype = BUS_VIRTUAL; // 虚拟总线
    uidev.id.vendor  = 0x1234;
    uidev.id.product = 0x5678;
    uidev.id.version = 1;

    // 设置坐标边界 (非常重要，否则内核会丢弃越界事件)
    uidev.absmin[ABS_MT_POSITION_X] = 0;
    uidev.absmax[ABS_MT_POSITION_X] = width;
    uidev.absmin[ABS_MT_POSITION_Y] = 0;
    uidev.absmax[ABS_MT_POSITION_Y] = height;
    uidev.absmin[ABS_X] = 0;
    uidev.absmax[ABS_X] = width;
    uidev.absmin[ABS_Y] = 0;
    uidev.absmax[ABS_Y] = height;
    uidev.absmin[ABS_MT_PRESSURE] = 0;
    uidev.absmax[ABS_MT_PRESSURE] = 255;

    write(uinput_fd, &uidev, sizeof(uidev));

    // 5. 向内核发送创建设备指令
    if (ioctl(uinput_fd, UI_DEV_CREATE) < 0) {
        LOGE("Failed to create uinput device");
        close(uinput_fd);
        return JNI_FALSE;
    }

    LOGI("Uinput device created successfully");
    return JNI_TRUE;
}

// 原子化事件注入接口
extern "C" JNIEXPORT void JNICALL
Java_com_yourapp_remotectrl_input_UinputInjector_nativeInjectEvent(JNIEnv *env, jobject thiz, jshort type, jshort code, jint value) {
    if (uinput_fd < 0) return;
    struct input_event ev;
    memset(&ev, 0, sizeof(ev));
    // 注意：这里不需要我们在应用层伪造时间戳
    // Linux 内核会在事件进入 input 子系统时，自动打上极度精准的高精度时间戳！
    ev.type = type;
    ev.code = code;
    ev.value = value;
    write(uinput_fd, &ev, sizeof(ev));
}

extern "C" JNIEXPORT void JNICALL
Java_com_yourapp_remotectrl_input_UinputInjector_nativeDestroy(JNIEnv *env, jobject thiz) {
    if (uinput_fd >= 0) {
        ioctl(uinput_fd, UI_DEV_DESTROY);
        close(uinput_fd);
        uinput_fd = -1;
    }
}
```

### 第二步：Java 层 JNI 封装 (`UinputInjector.java`)

在 `core/input-server/src/main/java/com/yourapp/remotectrl/input/` 创建 `UinputInjector.java`。它负责调用上述 C++ 代码，并保留你之前做好的**仿生学特征计算**：

Java

```
package com.yourapp.remotectrl.input;

public class UinputInjector {
    static {
        // 由于是独立运行的 jar，我们需要绝对路径加载 so 库
        System.load("/data/local/tmp/libuinput-ctrl.so");
    }

    private native boolean nativeInit(int width, int height);
    private native void nativeInjectEvent(short type, short code, int value);
    private native void nativeDestroy();

    private static final short EV_SYN = 0x00;
    private static final short EV_KEY = 0x01;
    private static final short EV_ABS = 0x03;
    
    // ... 常量定义与之前相同 (ABS_MT_POSITION_X, BTN_TOUCH 等) ...

    private int trackingIdCounter = 1;
    private final java.util.Random random = new java.util.Random();

    public boolean init(int screenWidth, int screenHeight) {
        return nativeInit(screenWidth, screenHeight);
    }

    private void sendEvent(short type, short code, int value) {
        nativeInjectEvent(type, code, value);
    }

    private int getNextTrackingId() {
        return (trackingIdCounter++) % 10000 + 1;
    }

    public void injectTap(int x, int y) throws Exception {
        int trackingId = getNextTrackingId();

        // 1. DOWN
        sendEvent(EV_ABS, ABS_MT_SLOT, 0);
        sendEvent(EV_ABS, ABS_MT_TRACKING_ID, trackingId);
        sendEvent(EV_ABS, ABS_MT_POSITION_X, x);
        sendEvent(EV_ABS, ABS_MT_POSITION_Y, y);
        sendEvent(EV_ABS, ABS_MT_PRESSURE, 50 + random.nextInt(10));
        sendEvent(EV_ABS, ABS_MT_TOUCH_MAJOR, 10);
        sendEvent(EV_KEY, BTN_TOUCH, 1);
        sendEvent(EV_KEY, BTN_TOOL_FINGER, 1);
        sendEvent(EV_SYN, SYN_REPORT, 0); // 发送 SYN 后，C++ 层的单次 write 事件被内核打包处理

        Thread.sleep(40 + random.nextInt(20));

        // 2. UP
        sendEvent(EV_ABS, ABS_MT_SLOT, 0);
        sendEvent(EV_ABS, ABS_MT_TRACKING_ID, -1);
        sendEvent(EV_KEY, BTN_TOUCH, 0);
        sendEvent(EV_KEY, BTN_TOOL_FINGER, 0);
        sendEvent(EV_SYN, SYN_REPORT, 0);
    }

    public void destroy() {
        nativeDestroy();
    }
}
```

### 第四步：调整主应用的 Payload 释放逻辑 (`InputInjector.kt`)

这是关键的一步。由于引入了 `.so` 库，你的主应用（在 `core/root` 模块下的 `InputInjector.kt`）在启动幽灵进程时，不仅要复制 `jar`，还要把编译好的 `libuinput-ctrl.so` 一起复制到 `/data/local/tmp/`。

修改 `InputInjector.kt`：

Kotlin

```
    companion object {
        private const val PAYLOAD_JAR = "input-server.jar"
        private const val PAYLOAD_SO = "libuinput-ctrl.so" // 新增
        private const val TARGET_JAR_PATH = "/data/local/tmp/input-server.jar"
        private const val TARGET_SO_PATH = "/data/local/tmp/libuinput-ctrl.so"
    }

    private fun deployAndStartPayload() {
        try {
            val assetManager = context.assets
            // 复制 JAR
            val cacheJar = File(context.cacheDir, PAYLOAD_JAR)
            assetManager.open(PAYLOAD_JAR).use { input ->
                FileOutputStream(cacheJar).use { output -> input.copyTo(output) }
            }
            // 复制 SO
            val cacheSo = File(context.cacheDir, PAYLOAD_SO)
            assetManager.open(PAYLOAD_SO).use { input ->
                FileOutputStream(cacheSo).use { output -> input.copyTo(output) }
            }

            Shell.cmd(
                "cp ${cacheJar.absolutePath} $TARGET_JAR_PATH",
                "cp ${cacheSo.absolutePath} $TARGET_SO_PATH", // 移动 SO 库
                "chmod 777 $TARGET_JAR_PATH",
                "chmod 777 $TARGET_SO_PATH"
            ).exec()
```

### 总结

完成上述改造后，你的项目实现了跨时代的演进：

当你在远端点击屏幕时，数据流向变为 `Socket -> Java JNI -> C++ -> /dev/uinput`。

Linux 内核在收到 `uinput` 的事件后，会认为这是**设备通过 USB 或内部总线新插上了一块名为 "SuperControler_Virtual_Touch" 的触控板**。内核会走标准的中断下发流程，自动补齐底层上下文并打上原生的系统时间戳，完美化解了直接操作裸节点带来的安全风险和不稳定因素。
