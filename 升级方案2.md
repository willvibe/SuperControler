**危害**：虽然目前 90% 的手机是 64 位内核，但一些老旧的备用机（例如某些 32 位系统或 32 位内核编译的 Android 10/11）、以及大多数 Android 模拟器，其内核的 `struct input_event` 中 `timeval` 是由两个 32 位的 `int` 组成的（即总长 16 字节）。如果在这些设备上强行写入 24 字节，内核驱动解析会直接错位，导致 `kernelFailCount` 瞬间涨满，回退到框架注入（打卡再次被发现）。

#### 1. 解决 32/64 位架构自适应问题

我们可以通过简单的判断（尝试往节点写一次数据并捕获异常，或者判断系统的架构属性）来决定发 16 字节还是 24 字节。最稳妥的方式是在 Java 层根据系统架构动态调整。

修改 `KernelTouchInjector` 的成员变量和 `sendEvent` 方法：

Java

```
// 动态判断是否为 64 位内核
private final boolean is64Bit = android.os.Process.is64Bit();
private final int structSize = is64Bit ? 24 : 16;
private final byte[] bufferArray = new byte[structSize];
private final ByteBuffer buffer = ByteBuffer.wrap(bufferArray).order(ByteOrder.LITTLE_ENDIAN);

// 补充老式单点触控常量
private static final short ABS_X = 0x00;
private static final short ABS_Y = 0x01;

private synchronized void sendEvent(short type, short code, int value) throws Exception {
    buffer.clear();
    long time = System.currentTimeMillis();
    long sec = time / 1000;
    long usec = (time % 1000) * 1000;

    if (is64Bit) {
        buffer.putLong(sec);
        buffer.putLong(usec);
    } else {
        buffer.putInt((int) sec);
        buffer.putInt((int) usec);
    }
    buffer.putShort(type);
    buffer.putShort(code);
    buffer.putInt(value);

    out.write(bufferArray);
}
```
